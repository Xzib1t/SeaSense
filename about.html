About

This web application has been created for plotting data gathered by the SeaSense sensor suite.

By default, all data contained within the CSV file will be plotted at the same time when loaded into the application. Sensor data can be toggled for display using the radio buttons for each sensor's data located above the top right corner of the plot.

For more information about the SeaSense Arduino Library, please visit this page.

FAQ

What format should my data be in?

This application is capable of reading .CSV files containing data such as the following:

Time,Temp,Depth,Cond,Light,Head,AccelX,AccelY,AccelZ,GyroX,GyroY,GyroZ
14:27:34,30.76,125,809,721,36,0,0,-10,24,-1,0
14:27:34,30.76,125,813,721,36,0,0,-10,18,-6,-2
14:27:34,30.27,125,816,721,35,0,0,-10,18,4,3

All data logged by the SeaSense sensor suite is stored on the internal SD card, and said data files can be loaded directly into this application.

Alternatively, one can also obtain CSV files from the SeaSense by entering sd_dd or sd_cat FILENAME.CSV into the SeaSense Bluetooth command line interface and copying the output data to a .CSV file.

My graph isn't showing the correct date and time

This application assumes that filenames follow the SeaSense's 8.3 datestamp naming convention. For example, a proper filename could be 16062000.CSV, corresponding to the 00th file recorded on 06/20/2016. Since filenames are used to determine the correct date that the file contents correspond to, an incorrectly formatted filename may cause issues.

Note that if a filename doesn't contain the date of the file's recording, the graph will default to showing the current date. If the filename passed is completely invalid, the graph will show all data corresponding to 12/31/1969 at 19:00:00.

The SeaSense can't recognize my SD card, what gives?!

Unfortunately, the SD card library that the SeaSense relies on is not the most robust. If you find that the SeaSense no longer recognizes your SD card, try one of the following solutions:

  • Make sure the SD card is fully inserted into the SeaSense's microSD slot
  • Perform a system reboot by cycling the SeaSense's power*
  • Reset the SeaSense with the SD card removed, and insert the SD card before the system initialization sequence has finished*
  • Connect to the SeaSense's Bluetooth command line client and re-initialize the SD card using the sd_init command
  • If the above doesn't work, try resetting the SeaSense using the reset command, or by manually pressing the reset button*
  • As a last resort, reformat the SD card to FAT16 or FAT32 format

*RESETTING OR REBOOTING WHILE WRITING TO THE SD CARD WILL CORRUPT IT
If your card is corrupted it must be reformatted to FAT16 or FAT32 format

How are you interpolating your data?

Data interpolation options are based on the d3.js line interpolation options. Documentation on each method can be found here.
